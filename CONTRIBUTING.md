## How Can I Contribute?

### Reporting Bugs

If you find a bug, please create an issue on GitHub with:

- A clear and descriptive title
- Steps to reproduce the bug
- Expected vs actual behavior
- The device and OS version you experienced the bug on

### Suggesting Enhancements

Have an idea for a new feature? I'd love to hear it! Open an issue describing:

- What the feature is
- Why it would be useful to most users
- Any potential alternatives you've considered

### Pull Requests

I'd love to see your pull requests! If you're working on a feature or bug fix:

1. Fork the repository
2. Create a new branch (`git checkout -b feature/your-feature-name`)
3. Make your changes
4. Test your changes to ensure they work and don't break existing functionality
5. Commit your changes (`git commit -m "Add some feature"`)
6. Push to the branch (`git push origin feature/your-feature-name`)
7. Open a Pull Request

## Coding Guidelines

- Follow the official [Dart style guide](https://dart.dev/guides/language/effective-dart/style)
- Keep UI code separated from business logic
- Write descriptive variable and function names
- Comment complex code logic
